# 비밀번호 관리 프로세스

## 📋 개요

KATC1 시스템의 비밀번호 관리는 **관리자 중심**으로 운영됩니다.
이메일 발송 기능 없이 관리자가 사용자 비밀번호를 초기화하고, 사용자가 직접 변경합니다.

---

## 👨‍💼 관리자 역할

### 1. 사용자 비밀번호 초기화

**경로:** 관리자 대시보드 → 사용자 관리 → 비밀번호 관리 탭

**절차:**
1. 관리자가 비밀번호 관리 탭 접속
2. 대상 사용자 선택
3. "비밀번호 초기화" 버튼 클릭
4. **시스템이 임시 비밀번호 자동 생성**
5. 화면에 표시된 임시 비밀번호를 사용자에게 전달 (메모, 메일, Slack 등)

**생성되는 임시 비밀번호:**
- 길이: 12자
- 구성: 대문자, 소문자, 숫자 혼합
- 예: `Abc123Xyz456`

### 2. 사용자에게 전달

⚠️ **주의:** 시스템에서 이메일로 자동 발송하지 않습니다.
관리자가 다음 중 하나의 방법으로 사용자에게 전달해야 합니다:

- 📧 이메일로 전달
- 📱 직접 전달 (대면/전화)
- 💬 메신저 앱 (Slack, Teams 등)
- 📝 문서/공지사항

---

## 👤 사용자 역할

### 1. 임시 비밀번호로 로그인

1. 로그인 페이지 접속: `/auth/login`
2. 이메일 입력
3. 관리자가 제공한 **임시 비밀번호** 입력
4. 로그인 완료

### 2. 비밀번호 변경 (필수)

⚠️ **중요:** 첫 로그인 시 반드시 비밀번호를 변경해야 합니다!

**경로:** 우측 상단 프로필 메뉴 → 비밀번호 변경

**절차:**
1. 이전 비밀번호 입력 (임시 비밀번호)
2. 새 비밀번호 입력
3. 비밀번호 확인 입력
4. "변경" 버튼 클릭
5. 변경 완료 후 새 비밀번호로 다시 로그인

**비밀번호 요구사항:**
- 최소 8자
- 대문자, 소문자, 숫자 포함
- 예: `MyPassword123`

## 🔄 전체 프로세스 다이어그램

```
[관리자]
  1. 사용자 관리 → 비밀번호 관리 탭 접속
  2. 사용자 선택 → "비밀번호 초기화" 클릭
  3. 임시 비밀번호 생성 (예: Abc123Xyz456)
  4. 임시 비밀번호 사용자에게 전달
           ↓
        (메일, 직접, 메신저 등)
           ↓
[사용자]
  1. 임시 비밀번호로 로그인
  2. 프로필 메뉴 → 비밀번호 변경
  3. 새로운 비밀번호 설정
  4. 새 비밀번호로 다시 로그인 (계속 사용)
```

---

## 🛡️ 보안 고려사항

1. **임시 비밀번호는 일회용입니다**
   - 사용자가 변경할 때까지 유효
   - 변경 후 이전 임시 비밀번호는 사용 불가

2. **비밀번호 전달 시 보안**
   - 암호화된 채널 권장 (메일, Slack 등)
   - 공개된 장소에서 전달하지 말 것

3. **사용자 비밀번호 분실**
   - 사용자가 비밀번호를 잊은 경우 → 관리자가 다시 초기화
   - 로그인 페이지의 "비밀번호 찾기" 기능은 현재 이메일 발송 없이 작동

---

## ❓ FAQ

**Q: 사용자가 비밀번호를 자꾸 잊어버립니다?**
- A: 관리자가 "비밀번호 초기화"를 다시 클릭하면 새로운 임시 비밀번호가 생성됩니다.

**Q: 이메일로 자동으로 임시 비밀번호를 보낼 수 없나요?**
- A: 현재는 SMTP 미설정으로 이메일 발송하지 않습니다. 향후 필요 시 SMTP 설정 후 기능 활성화 가능합니다.

**Q: 사용자가 "비밀번호 찾기"를 클릭하면?**
- A: 이메일이 없으므로 페이지만 표시되고 발송되지 않습니다. 대신 관리자에게 문의하도록 안내하세요.

**Q: 비밀번호 변경 후 이전 비밀번호는 사용 가능한가?**
- A: 아니요. 변경 후 새 비밀번호만 사용 가능합니다.
