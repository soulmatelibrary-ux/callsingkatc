# KATC1 인증 시스템 - 화면 구조 설계

**프로젝트**: KATC1 유사호출부호 경고시스템
**주제**: 사용자 인터페이스 및 화면 흐름 설계
**버전**: 1.0.0
**작성일**: 2026-02-19
**담당자**: AI Assistant

## 1. 화면 구조도 (Site Map)

```
KATC1 시스템
│
├─ / (포털 메인 페이지)
│  ├─ 상단 헤더
│  │  ├─ 로그인 버튼 (비로그인 상태)
│  │  └─ [사용자명] + 대시보드/로그아웃 (로그인 상태)
│  └─ 메인 콘텐츠 (시스템 소개)
│
├─ 인증 페이지 (로그인 전)
│  ├─ /login (로그인 페이지)
│  ├─ /signup (회원가입 페이지) ← 사전 등록으로 변경됨
│  ├─ /forgot-password (비밀번호 찾기)
│  ├─ /change-password (비밀번호 변경)
│  └─ /pending (승인 대기 페이지)
│
├─ 사용자 페이지 (로그인 후)
│  ├─ /dashboard (메인 대시보드)
│  │  ├─ 사용자 정보 표시
│  │  ├─ 항공사 정보
│  │  ├─ 최근 활동
│  │  └─ [유사호출부호] 버튼
│  ├─ /airline (유사호출부호 경고 시스템)
│  │  ├─ airline.html 모울 (목업 파일)
│  │  ├─ 항공사 데이터 필터링
│  │  └─ 경고 목록 조회
│  ├─ /profile (프로필 관리)
│  └─ /settings (설정)
│
└─ 관리자 페이지 (role='admin')
   ├─ /admin (관리자 대시보드)
   │  ├─ 사용자 현황 통계
   │  ├─ 시스템 상태
   │  └─ 최근 로그
   ├─ /admin/users (사용자 관리)
   │  ├─ 목록 조회
   │  ├─ 사용자 검색/필터
   │  ├─ 사용자 추가 (단일/일괄)
   │  └─ 사용자 상세 정보
   ├─ /admin/users/bulk-register (일괄 등록)
   │  ├─ CSV 파일 업로드
   │  ├─ 수동 입력 폼
   │  └─ 등록 결과 확인
   ├─ /admin/approval (사용자 승인)
   │  ├─ 승인 대기 목록
   │  ├─ 승인 / 거절
   │  └─ 메일 발송
   ├─ /admin/access-control (접근 관리)
   │  ├─ 역할 관리 (admin/user)
   │  ├─ 상태 관리 (active/suspended)
   │  └─ 항공사별 권한 설정
   ├─ /admin/password-reset (비밀번호 초기화)
   │  ├─ 사용자 검색
   │  ├─ 임시 비밀번호 생성
   │  └─ 리셋 이력 조회
   ├─ /admin/audit-logs (감시 로그)
   │  ├─ 로그 조회 (필터링)
   │  ├─ 사용자별 활동 추적
   │  └─ 시스템 변경 이력
   └─ /admin/settings (관리자 설정)
      ├─ 비밀번호 정책 설정
      ├─ 항공사 관리
      └─ 알림 설정
```

## 2. 사용자 화면 흐름 (User Flow)

### 2.1 초기 접속 흐름

```
┌────────────┐
│ / (메인)   │
│ 헤더에     │
│ 로그인 btn │
└─────┬──────┘
      │ 로그인 클릭
      ↓
┌────────────────────┐
│ /login             │
│ (로그인 페이지)    │
│ - 이메일 입력      │
│ - 비밀번호 입력    │
│ - 로그인 버튼      │
└─────┬──────────────┘
      │ 제출
      ↓
┌─────────────────────────────┐
│ 서버 로직                    │
│ 1. 이메일 확인              │
│ 2. 비밀번호 검증            │
│ 3. JWT 토큰 생성            │
│ 4. 상태 확인                │
└──┬──────────────┬─────────┬─┘
   │              │         │
   ├─ pending     ├─ active ├─ suspended
   │              │         │
   ↓              ↓         ↓
/pending      /change-   ERROR
(대기)        password  (정지됨)
              (초기변경)
              또는
              /dashboard
              (활성)
```

### 2.2 초기 비밀번호 변경 흐름 (사전등록 사용자)

```
관리자가 사용자 등록
(임시 비밀번호 생성)
      │
      ↓
사용자 로그인
(임시 비밀번호 입력)
      │
      ↓
/change-password로 강제 리다이렉트
(forceChangePassword = true)
      │
      ├─────── 필수 정책 ───────┐
      │ - 8자 이상                │
      │ - 대문자 포함             │
      │ - 소문자 포함             │
      │ - 숫자 포함               │
      │ - 특수문자 포함           │
      │                           │
      └───────────────────────────┘
      │
      ↓
새 비밀번호 입력 + 확인
      │
      ↓
POST /api/auth/change-initial-password
      │
      ↓
/dashboard 자동 이동
(정상 로그인 상태)
```

**메인 페이지 - 로그인 후 버튼 동작**:
- **[유사호출부호]** 버튼
  - 클릭 시: `/airline` 페이지로 이동 (또는 airline.html 모달)
  - airline.html 파일은 현재 목업 파일 (향후 완전히 통합될 예정)
  - 사용자의 항공사 정보에 따라 해당 항공사 데이터만 필터링하여 표시
  - 권한: 로그인한 사용자만 접근 가능

## 5. 요소별 상세 명세

### 5.1 Header 컴포넌트

**비로그인 상태**:
- 좌측: 로고 + 시스템명
- 우측: [로그인] 버튼

**로그인 상태 (일반 사용자)**:
- 좌측: 로고 + 시스템명
- 우측: [사용자명] [대시보드] [로그아웃]

**로그인 상태 (관리자)**:
- 좌측: 로고 + 시스템명
- 우측: [사용자명] [관리자 페이지] [대시보드] [로그아웃]

### 5.2 사이드바 (관리자 전용)

**메뉴 구조**:
1. 대시보드 (대시보드 아이콘)
2. 사용자 관리 (사람 아이콘)
   - 목록 조회
   - 사용자 추가
   - 일괄 등록
3. 사용자 승인 (체크 아이콘)
4. 접근 관리 (잠금 아이콘)
5. 비밀번호 초기화 (열쇠 아이콘)
6. 감시 로그 (로그 아이콘)
7. 설정 (톱니 아이콘)

### 5.3 폼 검증 규칙

**로그인 폼**:
- 이메일: 필수, 유효한 이메일 형식
- 비밀번호: 필수, 최소 8자

**비밀번호 변경 폼**:
- 현재 비밀번호: 필수
- 새 비밀번호: 필수
  - 최소 8자
  - 대문자 1개 이상
  - 소문자 1개 이상
  - 숫자 1개 이상
  - 특수문자 1개 이상
  - 현재 비밀번호와 다름
- 비밀번호 확인: 새 비밀번호와 일치

```typescript

const { data } = useQuery({

    `/api/callsign-warnings?airline_id=${user.airline_id}`
  )
})
```
